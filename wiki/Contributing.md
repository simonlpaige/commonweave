# Contributing to Commonweave

There are four ways to contribute. Pick the one that fits.

---

## 1. Add organizations from your network

If you work with, are part of, or know about cooperative, mutual-aid, or commons-based organizations that are missing from the directory, we want them.

**The easiest path:** Open a GitHub issue with the title "Organization addition: [Name]" and include:
- Organization name
- Website (if any)
- Country
- City (optional but helps with geocoding)
- Brief description (1-2 sentences)
- Which framework area it fits (see [[Directory-Methodology]])

We prioritize additions from under-covered regions. If you're adding organizations from Africa, Southeast Asia, Latin America, Central Asia, or the Middle East, your additions will move the needle more than another European co-op.

**Bulk additions:** If you have a list (CSV, JSON, spreadsheet), open an issue and attach it. We'll write an ingest script.

---

## 2. Correct errors

If an organization in the directory has:
- Wrong country or city
- Wrong framework area
- Outdated or wrong website
- Wrong description
- Should be removed (dissolved, changed mission, mis-categorized)

Open a GitHub issue with the title "Correction: [Organization Name]" and describe what's wrong and what's right.

---

## 3. Contribute to the framework docs

The framework docs (README.md, RESEARCH.md, DEEP-DIVE.md) are living documents with honest "we don't know yet" sections.

Things that are welcome:
- Corrections to factual claims (especially legal/historical claims)
- Evidence that a section's claims are stronger or weaker than stated
- Missing examples (especially non-Western examples - the docs have a strong Anglo bias)
- Translations

Things that are not welcome at this stage:
- Major structural rewrites - the framework has a specific design and changing the core requires discussion first
- Adding new principles - open an issue to discuss before drafting

For corrections: open an issue or submit a pull request with the specific text change and a citation for the correction.

---

## 4. Contribute code

The pipeline is JavaScript and Python. The main needs:

**High value, well-scoped:**
- Geocoding improvements (current rate: 19% - Nominatim works but is slow)
- New data source integrations for under-covered regions
- Frontend improvements to `map.html` and `directory.html`
- Deduplication improvements

**Documentation:**
- This wiki (you're reading it)
- Code comments in `tools/` scripts

**Before submitting a PR:**
- Run `python tools/pipeline_auditor.py` and make sure no new proposals are generated by your change
- Keep code simple and readable - this is intentionally accessible, not engineered for scale

---

## What We Won't Accept

- Organizations that fail the inclusion criteria (see [[Directory-Methodology]])
- Automated bulk ingests without human review of at least a sample
- Changes to the safety layer without discussion (dedup logic, staleness detection, legibility tracking)
- Personal data or contact information for individuals

---

## Identity

Contributors don't need to use real names. Pseudonymous contributions are fine. The project won't ask you to unmask.

If you're contributing from inside an institution that might not look kindly on it, use whatever operational security you need. Use a ProtonMail address, keep timezone metadata clean, commit under a handle. The work matters more than the attribution.

---

## Code of Conduct

Short version: be specific, be honest, don't punch at people. If you disagree with a decision, say so with evidence. The project is small enough that most things can be worked out in a GitHub issue.

Longer version: to be written when the community is big enough to need it.
